A1: Interactive Visualization

This visualization is designed to help users explore weather patterns across the United States and see how they change over time. Users can choose which weather metric they want to view using the metric dropdown, including average temperature, minimum temperature, maximum temperature, average wind speed, and fastest 5-second wind speed. The date slider allows users to move through the year and compare how these patterns change from day to day.

The choropleth map shows the selected weather metric for each state on the chosen date. The color of each state represents the magnitude of that metric, and the legend above the map helps users interpret the scale. Hovering over a state reveals the exact value for that state, which provides details on demand without cluttering the map.

The visualization also includes a linked line chart below the map. Clicking on a state updates the line chart to show how the selected metric changes over time for that specific state. This makes it possible to connect a single-day geographic snapshot on the map with a longer-term trend for an individual location. Users can also zoom and pan within the line chart to focus on smaller time periods in more detail.

Together, these interactions allow users to explore both spatial patterns across the country and temporal trends within individual states. The play button provides another way to move through time by automatically animating the slider, making it easier to observe how weather patterns shift over the course of the year.

Write-Up

Introduction

We used the Daily Weather in the U.S. dataset, which contains weather measurements collected from stations across the United States in 2017. The dataset includes variables such as minimum, maximum, and average temperature, along with several wind speed measurements. Because the raw dataset contains multiple station observations per state for each day, we aggregated the data by state and date to compute a state-level average for the selected metric. This allowed us to present the data in a way that is easier to compare geographically while still preserving meaningful variation over time.


In our visualization, users can explore several weather variables, including average temperature, minimum temperature, maximum temperature, average wind speed, and fastest 5-second wind speed. This subset of variables allows users to compare both temperature and wind conditions across states and observe how these patterns change throughout the year.

Design decisions

We chose a choropleth map as the main visualization because weather data is inherently geographic. A map makes it easier to identify regional patterns and compare nearby states. For example, users can quickly see north-south temperature differences or identify regions with relatively stronger wind conditions. We felt this approach provided more clarity and informational value than a more abstract chart.

We also considered using a graph-based visualization with adjustable variables, similar to the approach used in Lab 4. However, we felt that a scatterplot or general graph would not communicate the geographic nature of the weather data as effectively. It would also make it harder for users to understand regional patterns or locate a specific state quickly.

Using a map also helps reduce the gulf of evaluation. Instead of searching for a point in a dense graph, users can directly interact with familiar geographic shapes. Clicking on a state is a more intuitive way to retrieve detailed information about that location.

One key interaction we implemented is the dynamic date slider, which updates the map in real time as the user moves through the year. This supports temporal exploration and makes it easier to notice seasonal shifts or changes in regional patterns. We also added smooth transitions so users can better follow how the values change between dates.

Another important interaction is the linked line chart. When a user clicks on a state, the line chart updates to display that state’s trend over time for the selected metric. This linking allows users to move between an overview of the whole country and a more detailed view of one location. It also makes the relationship between geography and time much clearer.

To support more detailed exploration, we added zooming and panning in the line chart. This makes it easier to focus on a smaller part of the year without losing the larger context of the full trend. We also added tooltips on the map so users can view exact values on demand, as well as a dynamic annotation box that highlights key patterns visible in the current view.

We also considered using a calendar-style interface for date selection. While a calendar is useful for selecting one exact day, it is less effective for rapidly exploring many days in sequence. We felt that the slider was the better choice because it supports quick comparison across time and makes it easier to scan for changes in the data.

Finally, we kept a consistent color scale for each metric so that the same colors represent approximately the same values across different dates. This makes the legend more stable and helps users compare time periods more directly.

Development Process

Our development process involved building the visualization step by step, starting with loading and cleaning the dataset and then creating the base map of the United States using geographic data. After the map was working, we added the color encoding for the weather values and implemented the date slider so that the visualization could update dynamically.

In total, we spent approximately 10 people-hours developing the project. The most time-consuming aspects were cleaning and aggregating the dataset, implementing linked interactions between the map and line chart, and debugging issues related to updating multiple components simultaneously when the selected date or metric changed.

Work was divided among team members across several tasks including data preparation, map implementation, interaction design, line chart development, and write-up preparation. Aditya completed most of the initial data cleaning and aggregation and implemented the foundational choropleth map showing average temperatures. Emily implemented the linked state selection interaction, developed the line chart that displays temporal trends for the selected state, and added the play button animation for the date slider. All team members collaborated on writing and refining the project write-up.

Extra Credit Features

Our project includes several features that go beyond the base assignment requirements. One of these is the animated timeline playback controlled by the play button. Instead of manually moving the date slider, users can automatically animate the data over time. This makes it easier to observe how weather patterns evolve across the United States and helps reveal temporal trends that may not be immediately visible when viewing a single date.

We also implemented a linked multi-view visualization between the choropleth map and the line chart. When a user clicks on a state in the map, the line chart updates to display the full time series for that state. This interaction allows users to move seamlessly between spatial exploration (comparing states on the map) and temporal analysis (examining trends for a specific location).

Another additional feature is the dynamic annotation box, which updates based on the currently selected metric. The annotations provide contextual explanations about patterns visible in the visualization, helping guide users toward meaningful insights while they explore the data.

Finally, we implemented interactive zooming and panning within the line chart. This allows users to focus on smaller time intervals and explore detailed changes in weather patterns for a selected state. Combined with the linked map interaction, this supports both overview exploration and detailed analysis within the same interface.
