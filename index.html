A1: Interactive Visualization

This visualization allows users to explore weather patterns across the United States and see how they change over time. Users can select a weather metric using the dropdown menu, including average, minimum, and maximum temperature, as well as average wind speed and fastest 5-second wind speed. The date slider allows users to move through time and view how these patterns change from day to day.


The choropleth map displays the selected metric for each state on the chosen date. Colors represent the magnitude of the selected variable, with darker or warmer colors indicating higher values depending on the metric. The color legend above the map helps interpret these values. Hovering over a state shows the exact value for that state, and clicking on a state updates the line chart below the map. The line chart below the map displays how the selected metric changes over time for that specific state, allowing users to compare the map’s snapshot for a given day with the longer-term trend in that location. Together, these interactions allow users to explore both geographic patterns across the country and temporal trends within individual states.

Write-Up

Introduction

We used the Daily Weather in the U.S. dataset, which contains weather measurements collected from stations across the United States in 2017. The dataset includes variables such as minimum, maximum, and average temperature, as well as several wind speed measurements. Because the raw dataset contains multiple station readings per state for each day, we aggregated the data by state and date to compute a state-level average for the selected metric. This allowed us to visualize weather patterns across the country in a clearer and more interpretable way. In our visualization, users can explore several variables including average temperature, minimum temperature, maximum temperature, average wind speed, and the fastest five-second wind speed. This subset of variables allows users to compare both temperature and wind conditions across different states and observe how these patterns change over time.

Design decisions

We chose to use a choropleth map visualization combined with a dynamic date slider because we felt it provided the best clarity and informational value for this dataset. Since weather data is strongly tied to geography, a map makes it easy to see regional patterns and compare states visually. For example, users can quickly identify how temperatures differ between northern and southern states or observe areas with higher wind speeds.


We also considered using a graph-based approach with adjustable variables, similar to what was done in Lab 4. However, we felt that this approach did not capture the geographic nature of the dataset very well. A scatterplot or graph would make it harder to understand spatial relationships between states and would not clearly show how weather patterns vary across different regions of the country.


Using a map also helps reduce the gulf of evaluation. Users can directly click on the state they are interested in, rather than trying to locate a specific data point in a crowded chart. This makes the interaction more intuitive and lowers the effort required to retrieve information about a particular state.


One key interaction we implemented is a dynamic date slider that allows users to change the date and update the map in real time. This makes it easy to explore how weather conditions evolve over time. Smooth transitions help users track changes between dates and identify trends or shifts in the data as they move through the timeline.


Another interaction we created was a a panning and zooming effect for the line chart.

We also considered using a calendar-style interface for selecting dates. While calendars are a familiar way for users to choose a specific day, they are less effective for quickly exploring changes across many dates. The slider allows users to rapidly move through time and observe how weather patterns change, which better supports exploratory analysis. Because of this, we decided that the slider provided a better balance between usability and interactive exploration.


In addition to the map, users can click on a state to view a line chart showing how the selected weather metric changes over time for that state. This provides a more detailed view of temporal patterns and allows users to move between spatial and temporal perspectives in the dataset.

Development Process

Our development process involved building the visualization step by step, starting with loading and cleaning the dataset and then creating the base map of the United States using geographic data. After the map was working, we added the color encoding for the weather values and implemented the date slider so that the visualization could update dynamically.
